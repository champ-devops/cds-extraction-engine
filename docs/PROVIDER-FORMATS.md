# Transcription Provider JSON Formats

This document describes the JSON response formats from supported transcription providers and how they map to our internal TranscriptUtterance model.

---

## Internal Model (CoreAPI TranscriptUtterance)

```json
{
  "_id": "ULID",
  "transcriptID": "ULID",
  "speakerOriginal": "A",
  "speakerModified": null,
  "textOriginal": "The transcribed text of this utterance.",
  "textModified": null,
  "textOriginalSource": "AUTOGEN:ASSEMBLY",
  "textOriginalAt": "2025-01-31T12:00:00.000Z",
  "startMS": 2160,
  "endMS": 17200,
  "confidence": 0.889,
  "segmentIndex": 0,
  "deletedAt": null,
  "createdAt": "2025-01-31T12:00:00.000Z",
  "updatedAt": "2025-01-31T12:00:00.000Z"
}
```

### textOriginalSource Enum Values
| Value | Description |
|-------|-------------|
| `AUTOGEN:ASSEMBLY` | Auto-generated by AssemblyAI |
| `AUTOGEN:DEEPGRAM` | Auto-generated by DeepGram |
| `AUTOGEN:REVAI` | Auto-generated by Rev.ai |
| `HUMAN:CAPTIONER` | Human-generated caption file (generic) |
| `HUMAN:CAPTIONER_NAME` | Human-generated caption file (named captioner, e.g., `HUMAN:3PLAYMEDIA`) |
| `HUMAN:CUSTOMER_INSERTED` | Manually inserted by customer via UI |

```json
    "max_speakers_expected": 16

```

### Field Mapping: AssemblyAI → TranscriptUtterance

| AssemblyAI Field | TranscriptUtterance Field | Notes |
|------------------|---------------------------|-------|
| `utterances[].speaker` | `speakerOriginal` | Letter identifier (A, B, C, etc.) |
| `utterances[].text` | `textOriginal` | Full utterance text |
| `utterances[].start` | `startMS` | Already in milliseconds |
| `utterances[].end` | `endMS` | Already in milliseconds |
| `utterances[].confidence` | `confidence` | 0-1 float |
| (array index) | `segmentIndex` | 0-based index in utterances array |
| - | `textOriginalSource` | Set to `AUTOGEN:ASSEMBLY` |

### Important Notes
- **Time units**: AssemblyAI provides times in **milliseconds** (no conversion needed)
- **Speaker labels**: Letters (A, B, C...) - may need mapping to actual names later
- **Provider job ID**: Use `id` field to store in Transcript.providerJobID

### providerMeta Extraction

Store the following in `providerMeta` (excludes text/words/utterances):

```json
{

  "confidence": 0.9082543,
  "audio_duration": 9757,
  "punctuate": true,
  "format_text": true,
  "dual_channel": false,
  "speaker_labels": true,
```

**Excluded from providerMeta:** `text`, `words`, `utterances` (stored separately)

### Field Mapping: DeepGram → TranscriptUtterance

| DeepGram Field | TranscriptUtterance Field | Notes |
|----------------|---------------------------|-------|
| `results.utterances[].speaker` | `speakerOriginal` | Integer (0, 1, 2...) - convert to string |
| `results.utterances[].transcript` | `textOriginal` | Full utterance text |
| `results.utterances[].start` | `startMS` | **SECONDS** - multiply by 1000 |
| `results.utterances[].end` | `endMS` | **SECONDS** - multiply by 1000 |
| `results.utterances[].confidence` | `confidence` | 0-1 float |
| (array index) | `segmentIndex` | 0-based index in utterances array |
| - | `textOriginalSource` | Set to `AUTOGEN:DEEPGRAM` |

### Important Notes
- **Time units**: DeepGram provides times in **seconds** (multiply by 1000 for milliseconds)
- **Speaker labels**: Integers (0, 1, 2...) - convert to string for storage
- **Provider job ID**: Use `metadata.request_id` to store in Transcript.providerJobID
- **Utterances require `utterances=true`** in the request query string
- **Speaker diarization requires `diarize=true`** in the request query string

## SRT Caption Format

### File Structure
SRT (SubRip) files have a simple text-based format:

```
1
00:00:02,160 --> 00:00:17,200
Spring Hill Board of Mayor and Aldermen
will come to order.

2
00:00:17,500 --> 00:00:25,800
Let the record show it's July 21st
at 6:00 p.m.

```

### Parsing Rules
1. **Sequence number**: First line of each block (1, 2, 3...)
2. **Timecodes**: Format `HH:MM:SS,mmm --> HH:MM:SS,mmm`
   - Hours, minutes, seconds, milliseconds
   - Separator is comma (not period)
3. **Text**: One or more lines of subtitle text
4. **Blank line**: Separates blocks

### Field Mapping: SRT → TranscriptUtterance

| SRT Field | TranscriptUtterance Field | Notes |
|-----------|---------------------------|-------|
| (none) | `speakerOriginal` | Set to `UNKNOWN` unless parsed from text |
| Text lines (joined) | `textOriginal` | Join multiple lines with space |
| Start timecode | `startMS` | Parse and convert to milliseconds |
| End timecode | `endMS` | Parse and convert to milliseconds |
| - | `confidence` | Set to `1.0` (human-generated assumed accurate) |
| Sequence number - 1 | `segmentIndex` | Convert to 0-based index |
| - | `textOriginalSource` | Set to `HUMAN:CAPTIONER` or `HUMAN:CAPTIONER_NAME` |

### SRT Timecode Conversion
```javascript
// "00:02:34,567" → 154567 (milliseconds)
function parseTimecode(timecode) {
  const [time, ms] = timecode.split(',');
  const [hours, minutes, seconds] = time.split(':').map(Number);
  return (hours * 3600 + minutes * 60 + seconds) * 1000 + parseInt(ms, 10);
}
```

### Speaker Detection in SRT
Some caption files include speaker indicators in the text:
```
1
00:00:02,160 --> 00:00:17,200
[Mayor Johnson]: The meeting will come to order.

2
00:00:17,500 --> 00:00:25,800
[Alderman Smith]: I move to approve the minutes.
```

Parsing strategy:
1. Check for patterns like `[Speaker]:`, `SPEAKER:`, `>>Speaker:`, etc.
2. Extract speaker name and store in `speakerOriginal`
3. Remove speaker indicator from text for `textOriginal`

## VTT (WebVTT) Caption Format

### File Structure
VTT files are similar to SRT but with different header and timecode format:

```
WEBVTT

00:00:02.160 --> 00:00:17.200
Spring Hill Board of Mayor and Aldermen
will come to order.

00:00:17.500 --> 00:00:25.800
Let the record show it's July 21st
at 6:00 p.m.
```

### Key Differences from SRT
| Aspect | SRT | VTT |
|--------|-----|-----|
| File header | None | `WEBVTT` required |
| Sequence numbers | Required | Optional |
| Millisecond separator | Comma (`,`) | Period (`.`) |
| Cue settings | Not supported | Supported (positioning, etc.) |

### Field Mapping
Same as SRT, but use period for millisecond parsing.

## Provider Detection

To auto-detect which provider generated a JSON file:

```javascript
function detectProvider(json) {
  // AssemblyAI: has 'acoustic_model' and 'language_model' at root
  if (json.acoustic_model && json.language_model) {
    return 'ASSEMBLY';
  }
  
  // DeepGram: has 'metadata' and 'results' structure
  if (json.metadata && json.results && json.results.channels) {
    return 'DEEPGRAM';
  }
  
  // Rev.ai: has 'monologues' array
  if (json.monologues && Array.isArray(json.monologues)) {
    return 'REVAI';
  }
  
  return 'UNKNOWN';
}
```

## Configuration

Provider API keys should be stored in the application configuration:

```json
{
  "transcription": {
    "assemblyai": {
      "apiKey": "YOUR_ASSEMBLYAI_API_KEY",
      "baseUrl": "https://api.assemblyai.com/v2"
    },
    "deepgram": {
      "apiKey": "YOUR_DEEPGRAM_API_KEY",
      "baseUrl": "https://api.deepgram.com/v1"
    }
  }
}
```

These should be placed in the `FULL_CONFIG_JSON64` secret as described in `project.mdc`.
